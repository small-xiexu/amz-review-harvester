# 亚马逊评论慢速采集助手

这是一个本地浏览器扩展原型。它复用用户手动登录的亚马逊页面，在可见浏览器中慢速翻页，保存公开评论，并在采集结束后导出 Excel 数据表和 AI 分析报告。

## 安装

1. 双击项目内的 `打开扩展安装页面.cmd`。也可以手动打开 Chrome 的 `chrome://extensions`，或 Edge 的 `edge://extensions`。
2. 开启右上角的“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本项目文件夹：`C:\Users\Administrator\Documents\亚马逊评论下载`。

已经加载过旧版本时，请在扩展管理页面点击本扩展的“重新加载”，确保版本显示为 `0.8.9`。

## 使用

1. 手动登录亚马逊买家账号。
2. 点击浏览器工具栏中的扩展图标。
3. 输入一个或多个商品 ASIN、勾选采集站点，并设置等待时间和每个任务本次最多加载批次。多个 ASIN 可用换行、空格或逗号分隔；采集站点支持加拿大、美国、墨西哥任意组合。
4. 点击“开始 / 继续”。工具会自动打开对应站点的评论页，并持续点击 `Show 10 more reviews` 加载评论；多个 ASIN 与多个站点会组合成任务队列，按顺序自动采集。
5. 在 `AI Agent` 区块选择 OpenAI 或 Claude，填写接口地址、模型和 API Key，并点击“测试连接”。测试连接会真正调用一次模型并检查是否有回复；OpenAI 兼容接口支持自动探测 `/v1`，例如填写 `http://127.0.0.1:8888` 时，工具会自动尝试 `http://127.0.0.1:8888/v1`。高级提示词默认内置，也可以手动修改。配置会保存在浏览器本地，不会随“清空数据”一起删除。
6. 采集完成后，点击“导出 Excel”下载 `.xlsx` 数据表。工具会先补齐缺失翻译；翻译过程中会分段写回本地，避免中途失败导致已翻译内容白跑。Excel 可重复导出，也可以在 AI 报告生成中导出，不会让已有报告失效。
7. 需要分析时，点击“生成 AI 报告”。AI 报告生成完成后，点击“下载 AI 报告”下载 `.zip`，里面包含 HTML 分析报告。

扩展会保存进度并自动去重。刷新页面或关闭弹窗不会丢失已经采集的数据；未点击开始前输入的 ASIN、站点和采集参数也会作为草稿自动保存，下次打开弹窗会恢复。

运行中等待下一次加载时，状态卡会显示实时倒计时，例如“距离自动点击还有 84 秒”。

导出列顺序为：ASIN、站点、评论ID、评论地区、原始评论日期、评论日期、原始评星、评星、情绪、评论人、评论人主页、是否验证购买、Helpful数量、是否有买家实拍、图片数量、图片链接、是否有视频、评论产品的属性、颜色、尺寸、英文标题、英文评论、标题中文翻译、评论中文翻译、评论链接。评论日期会尽量转换为 `YYYY-MM-DD` 格式，例如 `2026-05-07`；评星会导出为数字格式，例如 `5.0`；是否验证购买统一为 `是`/`否`；Helpful 数量无值时导出为 `0`；图片链接会尽量去掉缩略图尺寸参数；无法识别时保留原文。

导出时会自动剔除非 Verified Purchase 评论，只保留已验证购买评论。

Excel 和 AI 报告已拆分导出：Excel 包含冻结表头、筛选、列宽、自动换行和可点击链接；AI 报告以 `.zip` 下载，里面包含 HTML 分析报告，重点呈现痛点、亮点、改品建议和 Listing 卖点。

## 风险边界

- 降低速度不能保证账号不受限制。
- 工具不会绕过验证码、访问限制或登录验证。
- 检测到验证页时会自动暂停。
- 建议使用较小页数、较长间隔，并仅采集确有业务需要的数据。
- 使用者需要自行确认其使用方式符合亚马逊条款和所在地法律。

## 当前支持

北美三站：美国、加拿大和墨西哥。

当前版本依赖亚马逊页面结构。如果页面结构变化导致无法识别评论，扩展会暂停而不会持续翻页。若某个站点触发重新登录，扩展也会自动暂停，登录后点击“开始 / 继续”即可接着跑。

中文翻译在导出 Excel 或生成 AI 报告时调用在线翻译服务。翻译进度会分段写回本地；翻译失败不会影响原始评论导出，失败的翻译列会留空。
